Throughout the course, you have learned various web development techniques, from HTML & CSS to JavaScript, as well as design theories like spacing and positioning. Now you have all the tools you need to make a website. Your goal for this project is to make any kind of website you want. It's all up to your creativity!

Requirements

This is an individual project.

You must create a fully functional website. By fully functional, we only mean fully functional on the front-end. This means that you might not have logins, signups, or commenting features that require back-end implementation, but everything else should work, i.e. when you click on any button or link, it should do something.

To see examples of past student projects, click here.

Views

You must create at least two (2) views. This may not necessarily be two pages: You can have a one-page layout, with at least 2 separate sections, each with different content and layout.

Features

Your project will need two (2) features of some significance. We want them to serve some purpose on your website, rather than being placeholders because we ask you to put them there. Each feature can range from unique animations, interactive things done in JavaScript. We encourage you to diversify these features; pseudo-selector like :hover effects will only count as one feature. Examples of such features include:

  • Pseudo-selectors -- don't overdo this as we count them once 🔮
  • A cool CSS animation -- something beyond a bouncing basketball? You may have a loading animation 🎡
  • An image carousel or slider -- you may code it yourself from scratch or learn an existing JavaScript library to achieve this 🎠
  • An interactive button in JavaScript (perhaps a play-now button that takes the player to the game mode from the menu page, note that a button taking the player back to the menu from can count as a different feature) -- some folks made games as their final projects 👾
  • Something in 3D if you're feeling experimental -- think of a rotating wireframe ball/cube spinning on your page?! 🎁
  • Anything else you can think of 💡

If you're unsure if your ideas will count towards these features, come talk to WDD staff before/during the project check-in 😃

Responsive Design

  • Your website must support following web browsers and screen resolutions,

    • Web Browser: Chrome, Safari, Firefox

    • Screen Resolution: 1024x768 ~ 1440x900;
      even better if the page layout's mobile compatible (optional)

  • As long as you stick to these requirements, you can make anything you want!

Write-up

Please create a separate PDF document that lists out and describes your two (2) most significant features (like a treasure map 🗺 so we can find them). Please also tell us about the significance and purpose of each of those features.

You'll only need to write a few short sentences about each feature, and finally attach this write-up to the project submission form.

Grading

  • Check-in: 4 points

    Just like the group project, there will be a check-in with the staff to review your current progress and give you feedback on your final project. Please come prepared with mockups for each of your pages. If you do not have mockups for both views and/or you do not check-in your project on time, you will not receive full credit. We highly encourage you come with some HTML skeleton code, but it is not required for the check-in.

  • UI Aesthetics: 10 points

    We will be judging the design of your website; in particular, spacing, color, typography & images, and positioning.

  • User Experience: 10 points

    We will also look at how the interface enhances the usability of your website. For example, if you have an image heavy website, grid UI might be better than using a straightforward list UI.

  • Significant features: 6 points

    There should be at least two (2) features on the website. They can be implemented using CSS and JavaScript. Again, only one pseudo-selector can count towards this requirement.

  • Write-up: 2 points

    As a friendly reminder, we ask that you identify these features in a separate write-up and explain their significant and purpose, similar to how you did in the group project. If you have any questions about what constitutes as a feature, let us know! Otherwise, Please attach this write-up as a PDF to the project submission form.

Total: 32 points

Using Online Resources

Here are some rules regarding the use of online resources. If you have a question of whether something is acceptable or not, don't hesitate to ask.

  • You are not allowed to use templates like premade themes for Wordpress and Tumblr.
  • You are not allowed to use CSS Libraries like Twitter Bootstrap or Foundation.
  • You are not allowed to use site builders like Squarespace, Wix, or WebFlow.
  • You are not allowed to copy the code of other websites and modify it.
    Trust us, we will know.
  • EXCEPTION: The use of JavaScript libraries (e.g. waypoints, masonry) is okay.

Design Process

Here's an example design approach you might want to take:

  1. Define the purpose of your website.
  2. Outline what content would go into your website.
  3. Research what UI would work the best for the type of information you are trying to convey.
  4. Create low-fi sketches.
  5. Create hi-fi design in Figma.
  6. Get feedback. (Check-in!)
  7. Create website with HTML, CSS and JavaScript.
  8. Complete your write-up describing the features you implemented!
